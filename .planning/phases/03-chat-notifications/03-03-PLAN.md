1. Create `lib/features/notifications/data/services/fcm_service.dart`:
   - Class `FcmService` with no constructor parameters (uses FirebaseMessaging.instance and FirebaseFirestore.instance directly)
   - Method `Future<void> init(String studentId)`:
     - Request notification permission: `await FirebaseMessaging.instance.requestPermission()` -- this triggers POST_NOTIFICATIONS dialog on Android 13+
     - Get initial token: `final token = await FirebaseMessaging.instance.getToken()`
     - If token is not null, call `_saveToken(studentId, token)`
     - Set up token refresh listener: `FirebaseMessaging.instance.onTokenRefresh.listen((newToken) => _saveToken(studentId, newToken))`
     - Set up foreground message handler: `FirebaseMessaging.onMessage.listen((RemoteMessage message) { _handleForegroundMessage(message); })`
   - Method `Future<void> _saveToken(String studentId, String token)`:
     - Save to `students/{studentId}/tokens/{token}` in Firestore: `{token: token, platform: 'android', updatedAt: FieldValue.serverTimestamp()}`
     - Use the token string as the document ID for deduplication
   - Method `void _handleForegroundMessage(RemoteMessage message)`:
     - Extract notification title and body from message.notification
     - Call LocalNotificationService.showNow() to display a local notification (since FCM does not show UI when foregrounded)
     - This method will be connected to LocalNotificationService in the provider wiring
   - Store a reference to LocalNotificationService (passed via init or set separately) for foreground display

   REVISED approach for cleaner architecture:
   - `FcmService` constructor takes `LocalNotificationService localNotificationService`
   - `init(String studentId)` does permission request + token save + listeners
   - Foreground handler calls `localNotificationService.showNow(title, body, payload)` to display notification when app is open

2. Create `lib/features/notifications/data/services/local_notification_service.dart`:
   - Class `LocalNotificationService` with `FlutterLocalNotificationsPlugin _plugin` field (initialized in constructor)
   - Method `Future<void> init()`:
     - Call `tz.initializeTimeZones()` from timezone package
     - Create `AndroidInitializationSettings('@mipmap/ic_launcher')`
     - Create `InitializationSettings(android: androidSettings)`
     - Call `_plugin.initialize(settings, onDidReceiveNotificationResponse: _onNotificationTap)`
     - Create notification channels:
       - Channel 'chat_messages' / 'Chat Messages' for chat notifications (importance: high)
       - Channel 'departure_reminders' / 'Departure Reminders' for departure reminders (importance: high)
   - Method `Future<void> scheduleDepartureReminder({required int id, required String rideOrigin, required String rideDestination, required DateTime departureTime})`:
     - Calculate reminder time: `departureTime.subtract(Duration(minutes: 15))`
     - If reminder time is before now, return early (already past)
     - Convert to TZDateTime: `tz.TZDateTime.from(reminderTime, tz.local)`
     - Call `_plugin.zonedSchedule(id, 'Ride departing soon', '$rideOrigin -> $rideDestination in 15 minutes', tzTime, notificationDetails, androidScheduleMode: AndroidScheduleMode.inexactAllowWhileIdle)`
     - Use 'departure_reminders' channel in AndroidNotificationDetails
   - Method `Future<void> showNow(String title, String body, {String? payload})`:
     - For foreground FCM display: show immediate notification via `_plugin.show(id, title, body, details, payload: payload)`
     - Use hashCode of payload or DateTime.now().millisecondsSinceEpoch for unique notification ID
     - Use 'chat_messages' channel in AndroidNotificationDetails
   - Method `Future<void> cancelReminder(int id)`:
     - Cancel a scheduled reminder: `_plugin.cancel(id)`
   - Method `void _onNotificationTap(NotificationResponse response)`:
     - Parse payload to determine navigation target
     - For now, log the tap -- navigation routing will be wired when GoRouter context is available
     - Can use a static callback or stream that the app listens to
