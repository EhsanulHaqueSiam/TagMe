2. Create `lib/features/rides/providers/schedule_providers.dart`:
   - `@riverpod` annotation with part directive

   **Provider: `mySchedulesProvider`**
   - StreamProvider that reads local profile ID via SharedPreferences
   - Calls `scheduleRepositoryProvider.getSchedules(studentId)`

   **Provider: `createScheduleProvider`**
   - Function provider that creates a RecurringSchedule from params: origin (lat, lng, address), destination (lat, lng, address), transportType, rideHailingTag, totalSeats, departureTime (HH:mm), daysOfWeek (List<int>)
   - Reads profile for studentId
   - Computes geohash using GeoFirePoint
   - Calls `scheduleRepositoryProvider.createSchedule(schedule)`

   **Provider: `processRecurringSchedulesProvider`**
   - Function that takes studentId, runs on app open (v1 client-side approach per RESEARCH.md Pattern 5):
     - Gets today's weekday: `DateTime.now().weekday` (1=Mon, 7=Sun)
     - Gets today's date string: `DateFormat('yyyy-MM-dd').format(DateTime.now())`
     - Calls `scheduleRepositoryProvider.getActiveSchedulesForToday(studentId)`
     - For each schedule where `lastPostedDate != todayStr` AND departure time is still in the future:
       - Calls routeService.getRoute for distance + polyline
       - Calls fareCalculator.calculateTotalFare for fare
       - Reads profile for poster info
       - Creates a Ride from schedule data + computed route/fare
       - Calls `rideRepositoryProvider.createRide(ride)`
       - Calls `scheduleRepositoryProvider.markPosted(schedule.id, todayStr)`
     - All wrapped in try-catch (non-fatal -- app continues even if auto-post fails)

   - Run build_runner after
