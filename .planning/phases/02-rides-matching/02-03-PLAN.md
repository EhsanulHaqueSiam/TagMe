2. Create `lib/features/rides/providers/search_providers.dart`:
   - Add `@riverpod` annotations and part directive

   **Provider: `nearbyRidesProvider`**
   - Stream provider that:
     - Reads user location from `currentLocationProvider`
     - If location is null, returns empty list stream
     - Calls `rideRepositoryProvider.ridesNearOrigin(userGeoPoint, radiusKm: 1.0)`
     - Maps results to sort by departureTime ascending (soonest first)

   **Provider: `myRidesProvider`**
   - Stream provider that:
     - Reads local profile ID from SharedPreferences (same pattern as nearby_students_provider)
     - If no profile ID, returns empty list stream
     - Calls `rideRepositoryProvider.myRides(profileId)`

   **Provider: `searchRidesProvider`**
   - Family provider taking search params: `({GeoPoint? destination, DateTime? departureTime, String? transportType})`
   - Stream provider that:
     - Reads user location from `currentLocationProvider` for origin geo query
     - If location null or destination null, returns empty list stream
     - Calls `rideRepositoryProvider.ridesNearOrigin(userGeoPoint, radiusKm: 1.0)` for origin proximity
     - Pipes results through `matchingServiceProvider.filterAndRankRides()` for destination proximity (1km), time window (+-30min), transport filter, and gender ranking
     - Reads user profile gender from profileProvider for gender ranking

   - Run build_runner after creating
